1. RESOLVED: DATABASE NOT-NULL CONSTRAINT ERROR (qte_num in pur.quotes)
--------------------------------------------------------------------------------
- File Modified:
  * app/Http/Controllers/PurchaseInitiationController.php

- Problem:
  When DProc or other users saved quotes, the frontend modal did not supply 
  a value for `qte_num`. Since the `pur.quotes` table has a NOT NULL constraint on 
  this column in the database, Laravel threw a database exception:
  "SQLSTATE[23502]: Not null violation: 7 ERROR: null value in column 'qte_num'"

- Fix Implemented:
  Added a robust backend check in the `add_quote` handler of the controller.
  If `$qteNum` is null or empty:
  * If updating an existing quote, it queries and preserves the existing `qte_num`.
  * If inserting a new quote, it automatically increments and assigns the next 
    sequence number by calculating `max(qte_num) + 1` for the active case (`pcs_id`).
